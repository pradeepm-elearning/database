EXPORT COMMAND 
--------------
1.Use only for version upgrade. For Example Dump taken from 10.2 version to 11.2 or higher version.
	expdp NEW_APP_OWNER_KIRAN/NEW_APP_OWNER_KIRAN@LP3ORA directory=DATA_PUMP_DIR dumpfile=NEW_APP_OWNER.dmp logfile=NEW_APP_OWNER.log version=10.2
2. Normal Export only with same version.
	expdp NEW_APP_OWNER_KIRAN/NEW_APP_OWNER_KIRAN@LP3ORA directory=DATA_PUMP_DIR dumpfile=NEW_APP_OWNER.dmp logfile=NEW_APP_OWNER.log

expdp NEW_APP_OWNER_KIRAN/NEW_APP_OWNER_KIRAN@CCDB schemas=NEW_APP_OWNER_KIRAN directory=DUMP_DIR dumpfile=NEW_APP_OWNER_KIRAN.dmp logfile=NEW_APP_OWNER_KIRAN.log

CHECK PERMISSION FOR THE DIRECTORY
-----------------------------------
SELECT * FROM all_tab_privs WHERE table_name = '<<DIRECTORY_NAME>>';


EXPORT:
1.    CREATE OR REPLACE DIRECTORY <<ORACLE-DIRECTORY-NAME>> AS '<<FOLDER-PATH>>';
e.g.: CREATE OR REPLACE DIRECTORY SCHEMA_DUMP AS 'F:\MY_ORA_DUMP';

2.	  GRANT CREATE ANY DIRECTORY TO <<SCHEMA_NAME>>;
e.g.: GRANT CREATE ANY DIRECTORY TO CC_TARGET_PT;

2.    GRANT READ, WRITE ON DIRECTORY <<ORACLE-DIRECTORY-NAME>> TO <<SCHEMA-NAME>>;
e.g.: GRANT READ, WRITE ON DIRECTORY SCHEMA_DUMP TO CC_TARGET_PT;

3.    expdp <<SCHEMA-NAME>>/<<SCHEMA-PASSWORD>>@<<DBNAME>> DUMPFILE=<<DUMP-FILE-NAME>>.DMP DIRECTORY=<<ORACLE-DIRECTORY-NAME>>;  ---------> Default Schema
e.g.: expdp CC_TARGET_ADMIN_R2_SIX/CC_TARGET_ADMIN_R2_SIX@SYS3 TABLES=K_MEANS,REGRESSION,FACTOR,ANLT_ANALYTICS_DEFINITIONS,WFL_DIAGRAMS directory=DATAPUMP_DIR dumpfile=abcde.dmp logfile=abcde.log reuse_dumpfiles=yes


IMPORT:
1.    SELECT * FROM ALL_DIRECTORIES;

2.    impdp <<SCHEMA-NAME>>/<<SCHEMA-PASSWORD>>@<<DBNAME>> DUMPFILE=<<DUMP-FILE-NAME>>.DMP DIRECTORY=<<ORACLE-DIRECTORY-NAME>>;
e.g.: impdp CC_TARGET_PT/CC_TARGET_PT@PTORA REMAP_SCHEMA=CC_TARGET_ADMIN_R2_SIX:CC_TARGET_PT directory=SCHEMA_DUMP dumpfile=DOMINOS.dmp logfile=DOMINOS.log
